Without AI vs With AI

MANUAL

// Developer writes tests by hand // 20 minutes later... test('password hashing', () => { const hash = hashPassword('test123'); expect(hash).toBeTruthy(); }); test('passwords match', () => { const hash = hashPassword('myPass'); expect(comparePassword('myPass', hash)) .toBe(true); }); // 2 tests. Passes. Ship it. // Missed: null, empty, boundary, // bcrypt format, error paths...
  • Slow — 20 min, 2 tests
  • Inconsistent — depends who writes it
  • Gaps — misses edge cases

WITH AI (e.g. Claude Code)

User: "Generate unit tests for authHelper.js" // AI generates tests in ~2 minutes: test('should hash valid password', ...) test('should reject 7-char password', ...) test('should accept 8-char password', ...) test('should throw on null input', ...) test('should not equal plaintext', ...) test('should match bcrypt format', ...) test('should handle bcrypt failure', ...) ... // 18 tests total // Boundary test at length=7 // caught off-by-one validation bug
  • Fast — 18 tests in 2 minutes
  • Systematic — covers more scenarios
  • Found real bug manual tests missed
AI is clearly better. But is just prompting AI enough?

Problem with Typical AI — SKILL Solves It

TYPICAL AI PROMPTING

// Every new session: User: "Write unit tests for hashPassword. Use AAA pattern. Use Jest. Test boundary values. Mock DB but keep bcrypt real. Name tests 'should X when Y'..." // Next session, different function: User: "Write unit tests for registerController. Use AAA pattern. Use Jest. Test boundary values. Mock DB but keep bcrypt real. Name tests 'should X when Y'..." // Same instructions every time.
  • Repetitive — re-teach every session
  • Inflated — stuff it all in system prompt,
       AI forgets halfway through
  • No memory — learns nothing from
       previous modules
  • No review — AI outputs code,
       no quality gate

SKILL FRAMEWORK

// Every session, one instruction: User: "Follow SKILL.md. Test auth module." // AI loads knowledge progressively: SKILL.md → Routes to correct agent ↓ agent.md → Plans (9 techniques) ↓ implementer → Writes code (AAA, Jest) ↓ reviewer → Verifies quality // Next module, AI reads past learnings: _memory/ → "Boundary tests caught bugs. Apply to next module."
  • One instruction — "Follow SKILL.md"
  • Progressive disclosure — loads ~500
       lines at a time, not 5,000
  • Memory — remembers what worked,
       applies to next module
  • Quality gate — reviewer checks
       before approving
Impact: Never repeat. Isolated context. Self-learning. Quality verified.

SKILL Framework

Progressive Disclosure

6 agents, each with plan → implement → review.

AI loads only the agent it needs. Never all 5,751 lines at once.

Each test type is isolated — unit tests don't pollute integration context.

Cleaner context, better output

Memory System

After testing auth, AI saves what worked to _memory/.

Next module reads it — "boundary tests catch off-by-one bugs."

AI gets smarter with each module. No re-teaching.

Self-learning across modules

Platform Portable

SKILL.md is just markdown steps. Any AI can follow them.

CLI: Claude Code, Codex (full auto)
Web+GitHub: ChatGPT, Gemini (read auto)
Web only: User pastes code

Same quality on any platform
